Chapter 1:

- token: la đơn vị xử lý ngôn ngữ của AI, đơn vị tính tiền khi sử dụng AI
	AI ko đọc text theo từng chữ hay từng từ - nó đọc theo token 
	vd: 
		"hello" -> 1 token
		"!" -> 1 token
		"hello world" -> 2-3 token 

	ở đây đối với tiếng anh sẽ ít tốn token nhất - vì những chữ khác nó kèm theo những kí tự khác 

	Quy ước nhanh: 100 token ~ 75 từ tiếng anh (tiếng khác có thể x1.5, 2.0 lần lên)

	việc gọi API -> nó cũng sẽ tính tiền theo token dựa trêm input + output -> lượng text nhập càng nhiều càng tốn token (output nó sẽ tốn nhiều token hơn input thường gấp 5-8 lần)

	những task đơn giản thì nên dùng các version: mini/flash/haiku thì nó sẽ rẻ hơn nhiều 


- Context window + khả năng ghi nhớ của AI
	là bộ nhớ tạm của AI trong 1 cuộc hội thoại (gần giống RAM của máy tính)
	-> khi gửi 1 request thì AI nó sẽ đọc hết all lịch sử trò chuyện 

	context window là giới hạn tổng số token AI có thể nhìn thấy trong 1 lần xử lý -> nếu vượt quá nó sẽ quên phần đầu cuộc trò chuyện (hiện tại thì gemini nó có context window tương đối lớn nhất)

	all chat từ đầu + all phản hồi + tài liệu, file, ... + system prompt nếu dùng API = tổng token tiêu thụ trên 1 lần gửi

	đóng tab/trình duyệt/xóa đoạn chat/tạo đoạn chat mới -> AI sẽ ko biết bạn là ai nữa

	1 số nền tảng có tính năng memory riêng giúp nhớ nhiều đoạn chat dù tạo mới nó vẫn nhận dạng được -> thì đây là 1 tính năng khác ko phải context window

	đối với developer dùng API: mình phải tự handle lịch sử nếu ko nó sẽ ko tự nhớ được 

**đối với nodejs -> cài thêm mvn để có thể quản lý và sử dụng nhiều version của nodejs**

chapter 5
mỗi cuộc hội thoại nó đều có giới hạn context window (input + output) -> sẽ đến lúc nó đầy -> mở 1 cuộc hội thoại mới (có thể nó sẽ nhớ 1 vài thông tin quan trọng của cuộc hội thoại khác "tự lưu vào knowledge của nó") nhưng nó sẽ ko lưu hết và đúng chắc chắn được

-> mỗi tính năng nên mở 1 cuộc hội thoại khác 
-> nên lưu những thông tin chung của dự án vào 1 file để nó đọc rules

đối với antigravity có 1 khái niệm là rules bộ nhớ vĩnh viễn của Agent 
-> mình sẽ tạo rules đây sẽ chứa những thông tin chung mà all task đều cần -> mỗi rq mình sẽ đính kèm theo rules là 1 file .md

Tạo rule trong antigravity:
	editor -> customizations -> rules có 2 phạm vi
		+workspace: có hiệu lực bên trong dự án đang làm
		+global: có hiệu lực cho tất cả dự án (thường viết những cái không được làm trong dự án)
		-> ở đây nếu rule ở workplace sẽ có độ ưu tiên cao hơn nếu bị conflict

ở đây cần tối ưu hóa phần rules vì mỗi rq đều kèm theo nó -> nên viết tối ưu nhất có thể tránh quá nhiều 

nên tạo nhiều file rule và set cho nó cái mode cụ thể: rules nào luôn luôn kèm theo, rule nào, để AI quyết, ...

Quy tắc:

	GEMINI.md: file router chính (kiểu khi nào viết file python thì đọc file python.md ...)
	python.md: rules riêng cho python

	testing.md: ... test
